## 2024-05-24 - Accessible Authentication and Interactive Divs
**Learning:** In vanilla HTML implementations without component libraries, form labels often get detached from their inputs visually relying solely on proximity, causing screen reader drop-off. Similarly, interactive visual elements like the "Cofre Selvático" are often built using `<div>` with `onclick` handlers but missing semantic button roles, keyboard event handlers (`onkeydown`), and `tabindex`, excluding keyboard-only users.
**Action:** Always ensure that `for` attributes are correctly linking `<label>` elements to input `id`s, especially in native dialog or modal forms. When converting `<div>` elements into interactive buttons, rigorously enforce the addition of `role="button"`, `tabindex="0"`, and keyboard event listeners (handling Enter/Space) to guarantee full accessibility.
